## 4. COMPONENT DETAILS

### ETL Pipeline Components
1. **APIExtractor**: Handles API rate limiting and implements a 3-attempt retry logic.
2. **DataTransformer**: Cleans missing values, validates types via Pydantic, and standardizes formats.
3. **DatabaseLoader**: Manages SQLite transactions, batch inserts (500 records/batch), and indexing.

### API Layer Components
1. **Health Router**: Provides `/health` for monitoring system status.
2. **Stations Router**: Manages metadata and statistics for the 44 available stations.
3. **Weather Router**: Handles filtered data retrieval and aggregate statistics.
4. **Dashboard Route**: Serves the static HTML/JS dashboard assets.

---

## 5. DATA FLOW

### ETL Data Flow
1. **Selection**: `StationSelector` identifies target station IDs.
2. **Extraction**: `APIExtractor` fetches raw JSON data.
3. **Transformation**: `DataTransformer` converts raw data into validated `WeatherDataModel` objects.
4. **Loading**: `DatabaseLoader` saves records to SQLite in optimized batches.

### API Request Flow
1. **Request**: User calls `GET /weather/?station_id=24`.
2. **Routing**: FastAPI routes the request to the handler.
3. **Injection**: `DatabaseLoader` is injected as a dependency.
4. **Query**: The loader executes an indexed SQL query.
5. **Serialization**: Results are validated and serialized to JSON.
